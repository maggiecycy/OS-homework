# 🐧 OS 系统知识速查 (Lab 1-2)

## 🛠️ 编译器命令 (gcc)
- `-g`: 包含调试信息。没有它，GDB 无法显示变量名和源码。
- `-Wall`: 开启所有警告。极其重要！能帮你抓住类型不匹配导致的“天文数字”错误。
- `-c`: 只编译，不链接。生成 `.o` 目标文件（零件）。
- `-o`: 指定输出文件名。

## 📜 系统调用 (System Calls)
- `open(path, flags)`: 向内核申请资源。返回 **fd (文件描述符)**。
  - 成功返回正数（通常从 3 开始），失败返回 -1。
- `read(fd, buf, count)`: 从内核缓冲区搬运数据。
  - 返回类型 `ssize_t`。返回 0 表示读到文件末尾（EOF）。
- `write(fd, buf, count)`: 将数据从用户空间甩给内核/设备。
  - 坑点：第 3 个参数应强转为 `(size_t)` 避免编译器溢出警告。
- `close(fd)`: 归还 fd 资源，防止进程因“代号不够用”而崩溃。

## 🐜 调试 (gdb)
- `b main`: 在程序入口拦截。
- `r`: 启动进程。
- `n`: 下一行 (Next)。赋值语句后，一定要 `n` 完才能 `p` 到结果。
- `p <var>`: 查看变量。**注意：是 fd 不是 df。**
- `q`: 遇到长分页时按 q 退出分页，进入 (gdb) 提示符。

## 🏗️ 自动化构建 (Makefile)
- `make`: 增量编译。提示 `up to date` 说明源码没变，无需重做。
- `make clean`: 物理清场。当逻辑产生“灵异现象”时，这是最高级的重启。
- **格式核心**: 命令前必须是 **Tab 键**，否则会报 `No rule to make target`。

## 🧠 OS 深度关联 (底层逻辑)
1. **资源抽象**: `fd = 3` 是因为 0, 1, 2 默认分给了键盘和屏幕。
2. **态势切换**: 每次执行 `read/write`，CPU 都会在“用户态”和“内核态”之间横跳。
3. **内存安全**: `ssize_t` 与 `size_t` 的区分，是 OS 保护内存不被“负数长度”撑爆的最后防线。

## 🏗️ 软件构建逻辑 (The Logic of Build)
1. **Source (.c)**: 指令集，决定程序“做什么”。
2. **Object (.o)**: 中间件，编译后的“零件”。
3. **Executable (./program)**: 产出物，最终跑在内核上的“实体”。
4. **Makefile**: 调度器。管理 1->2->3 的自动化过程，支持“改哪里，编哪里”。
